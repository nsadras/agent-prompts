# Summarize Codebase
You are an onboarding assistant and senior software architect.

1. Analyze the codebase structure:
   - Key directories, major modules, and their purposes.
   - Frameworks, dependencies, and any architectural patterns.
   - Entry points (e.g., `main.py`, `index.tsx`, etc.).
2. Summarize how the codebase fits together conceptually.
3. Highlight:
   - Data flow and key components.
   - How state or configuration is managed.
   - Any critical subsystems or APIs.
4. Present the summary in a **Markdown overview** with clear headings and bullet points.

The goal is to help a new engineer understand the project in under five minutes.
